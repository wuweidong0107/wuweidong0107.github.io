发表于 2020-06-11 分类于 程序人生 阅读次数：

一、高效地学习

推荐「费曼技巧」。

什么是费曼技巧？

如果有人用长篇大论跟你解释这个东西，那他实际上并不理解「费曼技巧」。

费曼技巧是一种「以教为学」的学习方式，能够帮助你提高知识的吸收效率，真正理解并学会运用知识。验证你是否真正掌握一个知识，看你能否用直白浅显的语言把复杂深奥的问题和知识讲清楚。

为什么它有效？

要做到能将复杂的问题用简单的语言说清楚，必须对知识有深刻的理解和应用。这需要你做到：

  1. 拆分和压缩知识。一本书很厚，里面的信息容量很大，我们无法记住所有的内容。但聪明的人会把书本呈现的信息进行压缩，提炼出规律和知识，来达到和原有的知识体系产生联系。压缩知识的过程，也是理解和内化的过程。

  2. 理解和简化知识。在理解和简化知识的过程中会不知不觉用到类比、举例、概括、对比等深度学习的方法。

  3. 理解和复述知识。讲不清楚，就是没理解。这也就是为什么有些软件开发的书籍让人读起来循序渐进深入浅出，而另一外一些书则让人头昏脑胀的原因：原作者本身对知识的理解程度太浅，以至于无法准确复述。

三、刻意练习

很多事情并不是有多难，而是没有去做。软件开发的很多知识并不是很复杂，很多时候只是要求开发人员熟练掌握如何应用一个技能，并没有要求进行伟大的发明与创造或者是探索某些未知领域。无法掌握新知识的原因只有一个：没有去学习并使用它，这里推荐一下「刻意练习」这个技巧。

什么是刻意练习？

本质是长时工作记忆。具有卓越才能的人，在进行钢琴、象棋等专业活动时，能够调用更大容量的工作记忆。通俗来说，就是他们拥有更大的内存。

这种长时工作记忆可以通过一定的练习来进行激活，通过一定难度的重复练习，在每次练习中收到反馈，不断纠正自己的错误，不断提升大脑的适应能力。

刻意练习的重点

  1. 刻意练习需要有好的导师.导师知道学习的最佳次序，理解并示范正确的方式来展示各种各样的技能，并可以提供有效的反馈。

  2. 学习分３个区域：舒适区 -> 学习区 -> 恐慌区。刻意练习主要发生在学习区，因为舒适区不能提高水平，而恐慌区难度过高。

  3. 刻意练习需要有效的反馈。反复做一件事情的目的在于找出不足，并进行改善和提高。不断进行自我检测，获得有效反馈，并及时进行改进和调整，类似于软件迭代。

我如何刻意练习 Markdown

如果我跟你说要坚持、要专注，那我就是一个大白痴，知道这2点的人没时间来阅读我的文章。所以我会提出一些更具体可行的建议。

  1. 将 Markdown 作为一个日常工具。比如将个人的学习 / 工作笔记可以以 Git + Markdown 的方式进行保存和管理，或者当你需要为你的代码写一份 README 时，为了读者的阅读体验，你应该用 Markdown。事实上，所有优秀的代码，都应该有一份详细的 README。

  2. 坚持每天用 Markdown 进行写作。写作类似于 Unix 系统里的标准输出 (stdin)，很多人无法写作的原因是因为他们还没有找到自己的标准输入 (stdiout)。如果你每天都在大量地读取一些珍贵的信息，这些宝贵的信息被你视为财富，那么你会自然而然且小心翼翼地找个保险柜将这些财富存储下来。所以写作不是目的，而是结果：你有太多需要存储下来的东西迫使你进行保存工作。向 Unix 系统学习，去找到合适你的标准输入吧。

  3. 本文就是一次刻意练习。
