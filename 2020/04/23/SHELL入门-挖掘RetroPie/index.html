SHELL入门_挖掘RetroPie

发表于 2020-04-23 阅读次数：

我在工作中写过一些 shell 脚本:

  • 测试稳定性的脚本;
  • 辅助定位 bug 的脚本；
  • 辅助日常工作的自动化脚本;
  • 用于快速构建软件原型的脚本;
  • 公司内部软件;
  • etc

也阅读过一些 shell 脚本的开源项目，例如：

  • RetroPie， 一个协助用户管理和安装复古游戏模拟器和相关软件的项目;
  • Armbian build， 一个用于制作嵌入式Linux发行版系统(Armbian)的项目;
  • etc

这些经历不断地促使我思考如何才能以正确的姿势使用 shell 脚本，事实上，我直径仍然在思索中、学习中。正确的姿势是指：

  • 确定什么场合用shell;
  • 如果编写出可维护、可扩展的shell脚本;
  • etc

我对 shell 脚本的定位是：

  • 一个可用、重要的脚本工具;
  • 该用用，不该用别用;
  • 要用的时候，得用得优雅。

回到正题，这篇文章是为了分享如何通过 RetroPie 来学习 shell 脚本。
